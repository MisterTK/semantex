The Problem

AI coding agents waste 60–80% of their tokens on search. Each grep→read loop adds to an ever-growing context window. The cost isn't linear — it's quadratic.

A single semantic search call replaces dozens of grep→cat→grep cycles, returning precisely the code that matters — with full context, in one shot.

How It Works

Index
AST-Aware Chunking

Tree-sitter parses 27 languages into semantic chunks — functions, classes, types — preserving structure.

Embed
ColBERT / PLAID

Per-token dense embeddings via ColBERT with PLAID indexing. MaxSim scoring for fine-grained matching.

Search
Hybrid Fusion

Dense + BM25 sparse search with query-adaptive convex combination. Best of both worlds.

Rank
Graph Propagation

Cross-file call graph and type references propagate scores for architectural queries.

Speed
17ms Latency

Warm searches in 17ms. Cold starts under 200ms. No GPU required.

Features

Privacy
Local-first

Your code never leaves your machine. All indexing and search happens locally with on-device models.

Integration
MCP Server

Works with Claude Code, Cursor, Windsurf, Cline, Continue, and Codex via the Model Context Protocol.

CLI
Terminal-native

Standalone CLI tool. Pipe results, integrate with scripts, use it anywhere you use grep.

Languages
27 Languages

Tree-sitter AST parsing for Rust, TypeScript, Python, Go, Java, C++, Ruby, Swift, and 19 more.

Search
Hybrid Retrieval

ColBERT dense embeddings + BM25 sparse search. Finds both exact identifiers and semantic concepts.
